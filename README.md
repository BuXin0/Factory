# 微信小程序：生产流程控制系统

## 1. 项目简介

本项目是一个纯前端（无后端依赖）的微信小程序，基于原生语法（`wxml / wxss / js / json`）实现生产流程控制，覆盖 10 步标准工艺流程：

1. 投料流程
2. 布兰德操作
3. 搅拌
4. 排气
5. 接IV
6. 蒸煮
7. 灭菌
8. 冷却
9. 接种
10. 等待包装

支持流程推进、参数实时监控、状态告警、操作日志记录，并使用本地缓存持久化状态。

## 2. 目录结构

```text
factory_101
├─ app.js
├─ app.json
├─ app.wxss
├─ project.config.json
├─ sitemap.json
├─ utils
│  ├─ constants.js      # 流程定义、参数阈值、缓存Key
│  └─ store.js          # 缓存读写、状态评估、日志记录
└─ pages
   ├─ home              # 首页（流程总览）
   ├─ process           # 流程页（分步骤控制）
   ├─ monitor           # 监控页（实时参数）
   └─ logs              # 日志页（操作记录）
```

## 3. 功能说明

### 首页（流程总览）
- 显示当前步骤、总进度、整体状态（正常/警告/异常）
- 可设置并保存当前操作员

### 流程页
- 展示 10 步流程导航与当前步骤检查项
- 支持“上一步/下一步”浏览步骤详情
- 仅允许完成“当前执行步骤”，已完成步骤不可回退

### 监控页
- 实时显示温度、压力、流量参数（2秒自动模拟波动）
- 支持手动拖动滑块修改参数进行测试
- 参数超阈值时自动进行红黄绿状态提示

### 日志页
- 自动记录每次操作事件：步骤、操作员、时间、参数、状态
- 支持清空日志

## 4. 本地缓存说明

使用 `wx.setStorageSync` / `wx.getStorageSync` 持久化数据：

- `factory_process_state`：流程状态、当前步骤、操作员、参数
- `factory_process_logs`：操作日志列表

可通过微信开发者工具 `Storage` 面板查看缓存内容。

## 5. 运行说明（微信开发者工具）

1. 打开微信开发者工具
2. 选择“导入项目”
3. 项目目录选择：`factory_101` 根目录
4. `AppID` 可选择测试号或使用项目内的 `touristappid`（仅本地调试）
5. 点击“编译”运行

## 6. 测试建议

1. 在首页输入操作员姓名并保存
2. 在流程页连续点击“完成当前执行步骤”，观察步骤推进
3. 在监控页拖动温度/压力/流量滑块到异常区间，观察颜色变化
4. 切到日志页，检查是否记录了参数更新和步骤操作
5. 关闭并重新打开小程序，确认流程状态和日志仍保留
